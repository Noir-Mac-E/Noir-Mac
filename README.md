## Overview

<a href="#">Noir</a> handles the web browsing use case that every other dark mode solution misses: the site you visit at 11pm that someone designed on a bright monitor and never considered would be read in a dark room. That light blast from a single tab in an otherwise dark browsing session is exactly the problem Noir was built to eliminate.

The <a href="#">per-site controls</a> let you configure different behavior for different websites. Disable Noir for a site that looks wrong with the transformation applied. Set a custom brightness level for a specific site that you visit frequently. <a href="#">Whitelist management</a> stores these per-site settings and applies them automatically on return visits without requiring manual reconfiguration.

<a href="#">Reading mode compatibility</a> works alongside Safari's built-in Reader view — Noir applies dark styling to Reader content as well as standard page content. Pages that are difficult to read in standard view and which you switch to Reader for benefit from consistent dark treatment. <a href="#">Video passthrough</a> leaves embedded video players unaffected by the dark transformation, preserving the intended appearance of video content while darkening the surrounding page.

The <a href="#">extension operates entirely on-device</a> without sending page content to external servers. The transformation algorithm runs locally in the Safari extension sandbox. No page content, browsing history, or site data leaves the browser. <a href="#">Low resource usage</a> applies transformations efficiently without measurable impact on page load times or battery consumption during extended browsing sessions.

## Additional Information

Noir's approach to per-site configuration reflects an understanding of how people actually browse. You don't want to configure dark mode per site manually across the entire internet — you want a default that handles the vast majority of cases automatically, with the option to override the handful of sites where the default doesn't look right. The whitelist system is built around that reality: you set it once per problematic site, and the exception is remembered permanently. The result is a tool that requires almost no ongoing maintenance once the initial week of visiting your regular sites has populated the settings.
